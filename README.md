## Overview

MazeRunner generates a seeded, weighted maze and runs multiple AI search algorithms through it simultaneously. Each algorithm is recorded step-by-step — every node it explores, the final path it finds, and how long it took. Results are exported to JSON and visualized in a JavaFX application, with charts generated via Python.

The project is split into three independent modules that communicate through a shared JSON interface.

---

## Architecture

```
MazeRunner/
├── engine/          ← Go — maze generation, all algorithms, JSON output
├── visualizer/      ← Java (JavaFX) — animation, results screen
├── analysis/        ← Python — matplotlib charts, batch testing, CSV export
└── data/
    ├── maze.json    ← generated by engine
    └── results.json ← generated by engine
```

### Data Flow

```
engine (Go)  →  maze.json + results.json  →  visualizer (Java)
                                          →  analysis (Python)
```

## Algorithms

| Agent | Type | Strategy |
|---|---|---|
| A\* (Manhattan) | Informed | g(n) + Manhattan heuristic |
| A\* (Euclidean) | Informed | g(n) + Euclidean heuristic |
| Greedy Best-First | Informed | Heuristic only, no cost tracking |
| UCS / Dijkstra | Optimal | Cumulative cost, no heuristic |
| BFS | Blind | Level-by-level exploration |
| DFS | Blind | Depth-first, no cost awareness |

## Terrain Weights

| Terrain | Value | Movement Cost |
|---|---|---|
| Wall | 0 | Impassable |
| Grass | 1 | 1x |
| Sand | 2 | 2x |
| Water | 4 | 4x |

## Prerequisites

### Engine (Go)
- [Go 1.23+](https://go.dev/dl/) — only needed if running without Docker
- [Docker](https://www.docker.com/products/docker-desktop/) — recommended, no Go install required

### Visualizer (Java)
- JDK 17+ with JavaFX
- Maven or Gradle
- [Gson 2.10.1](https://github.com/google/gson)

### Analysis (Python)
- Python 3.11+
- pip packages: `matplotlib`, `pandas`

## Getting Started

### 1. Clone the repository

```bash
git clone https://github.com/pkz074/mazerunner.git
cd mazerunner
```

### 2. Run the engine

**With Docker (recommended):**
```bash
docker compose build
docker compose run --rm engine
```

**Without Docker (requires Go):**
```bash
cd engine
go run cmd/main.go
```

This writes `data/maze.json` and `data/results.json`.

### 3. Run the visualizer

```bash
cd visualizer
# see visualizer/README.md for setup instructions
```

### 4. Run the analysis

```bash
cd analysis
pip install -r requirements.txt
python src/charts.py
```

## Engine CLI Flags

| Flag | Default | Description |
|---|---|---|
| `--seed` | `42` | Maze generation seed. Same seed = same maze |
| `--size` | `50` | Grid size (NxN) |
| `--agents` | `all` | Comma-separated agents or `all` |
| `--out` | `../data` | Output directory for JSON files |

### Examples

```bash
# Default run
docker compose run --rm engine

# Custom seed and size
docker compose run --rm engine --seed 99 --size 100 --agents all

# Specific agents only
docker compose run --rm engine --seed 42 --size 50 --agents astar_manhattan,bfs,dfs

# Without Docker
cd engine && go run cmd/main.go --seed 42 --size 100 --agents all
```

    "path_cost": 520,

]

## Adding a New Algorithm

1. Create a new file in `engine/internal/agents/`
2. Implement the `Agent` interface
3. Register it in `init()`

```go
package agents

import "github.com/pkz074/mazerunner/internal/maze"

func init() {
    Register(myAgent{})
}

type myAgent struct{}

func (a myAgent) Name() string { return "my_algorithm" }

func (a myAgent) Solve(m *maze.Maze) Result {
    // your implementation here
    return Result{}
}
```

No changes needed anywhere else.
